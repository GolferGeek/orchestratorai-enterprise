# PRD alignment

When this skill applies:

- Every **PRD** goal should trace to the **intention** and, for app work, to **Demo-grade minimums** in `docs/phase-00/intention-*.md` and **[DEMO-GRADE-BAR.md](../../docs/phase-00/DEMO-GRADE-BAR.md)** (product-shaped tools—not single static hero pages).
- **Non-goals** must be explicit so scope doesn’t creep during `/run-plan`.
- **Plans** must justify the PRD: milestones map to requirements; risks name what could break verification.
- Call out **drift**: if the plan introduces features not in the PRD, stop and update PRD or cut scope.
- Call out **weak demos**: if the plan would only deliver a starter screen, require revision to cover persistence, core loops, and testable workflows per the intention.
